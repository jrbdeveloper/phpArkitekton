Code Sample User Guide

Current Version: 1.8.0

The introduction of the new framework to the website is meant to isolate the various content sections from the programming logic used to present the site. This should make it easier to maintain the site. You will no longer need to worry about code that you don't need to understand, in this case PHP. At a minimum you should only need to understand how to manage content with HMTL and you should have a basic understanding of XML; no advanced knowledge is needed to manage the website inside of the framework. In the future the framework will be upgraded to include content management tools that will make it so that you can manage the site's content without having to use HTML and/or XML.

This document is structured to outline the various section of the sites homepage as well as the various page of the site itself. This particular website has two templates; one for the home page and one for all subpages. This is important to know when adding new pages to the website. How to do this is described in a later section of this user guide.

Recommendation for managing the files for the website

I would recommend not using the server file management tools and instead use something like Dreamweaver or some other web authoring application locally. Also, I would recommend using an FTP application to upload file to the server instead of the server file management tools.

Web Authoring Tool: Dreamweaver

FTP Client: CoreFTP

Once the content management system for the framework has been introduced this will no longer be an issue.

Managing Tour News

This section describes the process for managing the Tour News section of the home page. Located in the "pages" folder (/www/htdocs/ pages/) you will find an HTML file named tournews.html. This is the file that contains the content that is located in the Tour News section of the home page. To change the content that appears on the home page in this section just simply edit the content in the HTML file save your changes and then upload and overwrite what is on the server is the same location.

Keep in mind that this should only be the raw HTML for this section of the home page. To not enter the HTML you would expect to see in a regular HTML page. This HTML is created for you and rendered by the framework. By opening the tournews.html file you will see what I am talking about. This will also be the case when adding sub pages to the website. This process will be described in a later section of this user guide.

Managing Tour Course

This section describes the process for managing the Tour Course section of the home page. Located in the "data" folder (/www/htdocs/ content/data/) you will find an XML file named courses.xml. This is the file that contains the content that is located in the Tour Course section of the home page. To edit the content that appears on the home page for this section, simply edit the information in the xml file within the xml tags. Make sure the structure of the xml file is not changed in any way. Once complete save your changes and then upload the changes to the server to the appropriate location (/www/htdocs/ content/data/). The site also supports the ability to have multiple courses appearing on the home page. This can be accomplished by adding additional xml tag blocks as the example demonstrates.



Details

This is the containing tag the groups the course content sections together. As with all tags you should have a closing tag at the end of the section block. Example: <details>...</details>

 

Image

This tag represents the image that is located at the top of the content section "Tour Course" on the home page. As with all tags you should have a closing tag. Must begin with the forward slash.

 

Link

If it is desired that the image itself be clickable then this attribute would be used to specify the URL to take the user to when the image is clicked.

Height

This attribute is used to specify the height dimension of the image.

Width

This attribute is used to specify the width dimension of the image.

Alt

This attribute is used to specify the alt text that gets rendered in place of images if the user turns off image rendering in their browser.

Border

This attribute is used to specify the border around the image. If the Link attribute is used it is recommended that the border be set to 0 otherwise a link border will be added by default around the image.

Title

This tag represents the title that is located at the top of the content section "Tour Course" on the home page.

Content

This tag represents the content that is located in the "Tour Course" section on the home page.

Resource

This represents the content that is located in the "Tour Course" section on the home page. It will be laid out as such: For more information about [title] visit [resource]

title

This attribute represent the text that is clickable in the resource section.

Managing Pages

Adding Pages

This section describes the process for adding pages to the website. This is a multistep process that includes modifying an XML file to add new items to the menu as well as adding new page content to the page folder. I will document this in a step-by-step process.

Step 1. Open the menu.xml file located in the "navigation" folder (/www/htdocs/content/navigation/)

Step 2. Add an additional tag for the menu item link, title, keywords, description, template, page. Note the page tag should represent the .html file that you will be creating in the pages folder of the website (/www/htdocs/pages).

Step 3. Create a new HTML file and save it to the "pages" folder (/www/htdocs/pages/). This file should only include the HTML necessary for presenting the content in the format that you would like.

To see this process in action open the menu.xml file and study its structure and the corresponding HTML file to see its structure. Once the menu.xml file has been modified and the HTML files have been created you no longer have to worry about the index.php file. From that point it is only a matter of managing the content in the HTML file.

Editing Pages

This section describes the process for editing pages that exist in the website. It is assumed that you have already modified the menu.xml file and created the HTML file for the page that exists in the site. That being said, navigate to the pages folder (/www/htdocs/pages/) and open the file for the page that you want to modify. Make your changes, save the file and then upload it to the server.

Managing the Footer

This section describes the process for modifying the footer for the website. Located in the data folder (/www/htdocs/ content/data/) you will find the footer.xml file. The content is this file is presented on every page however you only need to modify it in this one location. Once your changes have been made to the page save them and upload the file to the server.
